# BZU Signal Bot 🤖

Автоматизований торговий бот для отримання сигналів BZU-USDT на OKX з Telegram-сповіщеннями.

## Що це робить

- ✅ **Кожні 15 хвилин** перевіряє технічні індикатори (EMA, RSI, Volume)
- ✅ **Автоматично** надсилає LONG/SHORT сигнали в Telegram
- ✅ **Розраховує** стоп-лос та тейк-профіти
- ✅ **Запобігає дублікатам** з системою cooldown
- ✅ **Обробляє помилки** та логує все для debugging

## Технічна аналізаー

### Умови для LONG
```
✓ EMA9 > EMA21         (тренд вгору)
✓ 35 < RSI < 60        (нейтральна зона)
✓ Ціна > EMA21         (цена вище опору)
✓ Обсяг > 1.2x         (підтвердження обсягом)
```

### Умови для SHORT
```
✓ EMA9 < EMA21         (тренд вниз)
✓ 35 < RSI < 60        (нейтральна зона)
✓ Ціна < EMA21         (ціна нижче опору)
✓ Обсяг > 1.2x         (підтвердження обсягом)
```

### Розрахунки
- **EMA (Exponential Moving Average)**: Швидка (9) та повільна (21) для тренду
- **RSI (Relative Strength Index)**: Період 14, діапазон 35-60
- **Volume**: Порівняння поточного обсягу із середнім за 20 свічок

## Налаштування

### 1. Налаштування GitHub Secrets

Перейдіть в **Settings → Secrets and variables → Actions** і додайте:

| Secret | Значення | Обов'язково |
|--------|----------|-----------|
| `TELEGRAM_TOKEN` | Ваш Bot Token від @BotFather | ✅ Так |
| `TELEGRAM_CHAT_ID` | Chat ID вашого чату | ✅ Так |
| `BALANCE` | USDT баланс для розрахунків (за замовчуванням 5) | ❌ Ні |
| `LEVERAGE` | Плече (за замовчуванням 20) | ❌ Ні |
| `INSTRUMENT` | Торговельна пара (за замовчуванням BZ-USDT-SWAP) | ❌ Ні |
| `BAR` | Таймфрейм: 1m, 5m, 15m, 1H, 1D (за замовчуванням 15m) | ❌ Ні |
| `EMA_FAST` | Швидка EMA період (за замовчуванням 9) | ❌ Ні |
| `EMA_SLOW` | Повільна EMA період (за замовчуванням 21) | ❌ Ні |
| `RSI_PERIOD` | RSI період (за замовчуванням 14) | ❌ Ні |
| `RSI_MIN` | Мінімальний RSI для входу (за замовчуванням 35) | ❌ Ні |
| `RSI_MAX` | Максимальний RSI для входу (за замовчуванням 60) | ❌ Ні |
| `VOLUME_MULT` | Мінімум обсягу мультиплікатор (за замовчуванням 1.2) | ❌ Ні |
| `RISK_PERCENT` | % ризику від балансу (за замовчуванням 0.20) | ❌ Ні |
| `STOP_LOSS_PERCENT` | % стоп-лосу (за замовчуванням 0.018) | ❌ Ні |
| `TP1_PERCENT` | % першого тейк-профіту (за замовчуванням 0.030) | ❌ Ні |
| `TP2_PERCENT` | % другого тейк-профіту (за замовчуванням 0.055) | ❌ Ні |
| `COOLDOWN_MINUTES` | Мінімум хвилин між однотипними сигналами (за замовчуванням 30) | ❌ Ні |
| `SEND_STATUS` | Надсилати статус без сигналів (за замовчуванням true) | ❌ Ні |

### 2. Отримання Telegram Bot Token

1. Напишіть [@BotFather](https://t.me/botfather) в Telegram
2. Команда `/newbot`
3. Слідуйте інструкціям, отримаєте token типу `123456789:ABCdefGHIjklmnoPQRstuvWXYZabcdefghi`

### 3. Отримання Chat ID

Напишіть боту будь-яке повідомлення, потім виконайте:
```bash
curl https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates
```

Знайдіть `"id"` у відповіді.

## Системи запобігання помилкам

### 1. Обробка мережевих помилок
- Timeout для OKX: 10 сек
- Retry логіка: Не розроблена (GitHub Actions перезапускає кожні 15 хв)
- Fallback: Напрацьовує безпечний вихід без сигналу

### 2. Cooldown система
Запобігає надсиланню однотипних сигналів частіше ніж кожні 30 хвилин.
Це дозволяє уникнути спамування та перепроцесів.

### 3. Валідація даних
- Перевіряє наявність мінімум 10 свічок
- Перевіряє валідність цін та обсягів
- Логує всі дивні значення

## Календар запусків

Bot запускається через GitHub Actions cron кожні 15 хвилин в UTC:

```
00:00, 00:15, 00:30, 00:45
01:00, 01:15, 01:30, 01:45
...и так далі
```

Якщо потрібно змінити частоту, відредагуйте `.github/workflows/signal.yml`:

```yaml
on:
  schedule:
    - cron: '*/15 * * * *'  # Кожні 15 хвилин
    # - cron: '*/5 * * * *'  # Кожні 5 хвилин
    # - cron: '0 * * * *'    # Кожну годину
    # - cron: '0 9 * * 1-5'  # Кожен робочий день о 9:00 UTC
```

## Статус-повідомлення

Якщо `SEND_STATUS=true`, бот надсилає коротке повідомлення в першу хвилину кожної години:

```
⏳ Немає сигналу  |  15:00 UTC
`BZ-USDT-SWAP` $100.50   RSI: 55.30
```

Це підтверджує, що бот працює та моніторить.

## Моніторування та аналіз

GitHub Actions зберігає логи для кожного запуску:

1. Перейдіть в **Actions → Latest workflow run**
2. Натисніть на job **check-signal**
3. Розгорніть секцію **Run signal bot** для перегляду логів

Логи зберігаються 90 днів за замовчуванням.

## Помилки та рішення

### ❌ Telegram повідомлення не приходять

1. **Перевірте Secrets**
   - `TELEGRAM_TOKEN` та `TELEGRAM_CHAT_ID` встановлені?
   - Чи коректні вони?

2. **Перевірте логи**
   - Actions → Latest run → Run signal bot

3. **Перевірте чат**
   - Чи бот мав доступ до чату?
   - `/start` команда виконана?

### ❌ Сигнали не генеруються

1. **Перевірте параметри RSI**
   - RSI_MIN та RSI_MAX встановлені реалістично?

2. **Перевірте обсяг**
   - VOLUME_MULT може бути занадто високим?

3. **Перевірте EMA**
   - EMA_FAST та EMA_SLOW правильно встановлені?

### ❌ GitHub Actions заблокований

GitHub обмежує рахунок на свободних планах. Якщо потрібно більше запусків, перейдіть на платний план.

## Бекап та безпека

- **Секрети**: Зберігаються шифровані в GitHub, не бачимо в коді
- **Логи**: Автоматично видаляються через 90 днів
- **Стан сигналів**: Зберігається в `/tmp` (видаляється при перезавантаженні)

## Документація модулів

### `analyzer.py`
- `ema(values, period)` — Експоненціальна ковзна середня
- `rsi(closes, period)` — Індекс відносної сили
- `volume_analysis(volumes, lookback)` — Аналіз обсягу
- `analyze_signal(...)` — Повна аналізація та генерація сигналу

### `okx_client.py`
- `OKXClient` — Клієнт для OKX API
- `get_candles(instrument, bar, limit)` — Отримання свічок

### `telegram_sender.py`
- `TelegramSender` — Надсилання повідомлень
- `send_signal(...)` — Надсилання торгового сигналу
- `send_status(...)` — Надсилання статусу

### `signal_state.py`
- `SignalState` — Управління станом сигналів
- `is_signal_allowed(signal)` — Перевірка cooldown
- `record_signal(signal)` — Запис останнього сигналу
